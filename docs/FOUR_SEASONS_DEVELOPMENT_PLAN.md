# 《四季山海远征》分阶段开发计划

关联规格：[FOUR_SEASONS_PRODUCT_SPEC.md](FOUR_SEASONS_PRODUCT_SPEC.md)

## 总体原则

- 每个阶段必须形成可单独试玩的版本，不积累到最后一次性交付。
- 每个阶段独立执行类型检查、单元测试、生产构建、桌面/Pad/手机真实浏览器验收。
- 新内容使用配置、策略或组件注册，禁止把季节和装备继续硬编码到 `Game`。
- 存档更新必须提供迁移和损坏数据回退。
- 移动端性能优先于桌面最高画质。

## 阶段 1：机械工坊与一张四季山海地图

目标：完成最重要的“看天气—改装—试驾—出发—获得反馈”闭环。

### 1.1 架构重构

- [x] 增加独立的 `BiomeDefinition`、`SeasonDefinition`、`WeatherId`，保留旧主题兼容层等待山海谷接入。
- [x] 增加 `SurfaceDefinition`：公路、泥地、沙地、浅水、深水、冰、深雪。
- [x] 增加 `MovementModuleStrategy`、`AmmoStrategy`、`ToolStrategy` 注册表。
- [x] 玩家实体改为装配式 `Loadout`，渲染器按行走模块和喷漆槽生成部件。
- [x] 存档升级到 v3，保存已解锁零件、三套方案、颜色和贴纸字段。
- [x] 增加 v2→v3 迁移测试与损坏集合回退。

### 1.2 3D 机械工坊

- [x] 新增工坊场景、转台、相机轨道和部件预览。
- [x] 实现拖动旋转、滚轮缩放、选择和点击安装。
- [x] 实现机械臂、磁吸定位光、火花和安装完成反馈。
- [x] 实现行走模块、炮弹、工具、外观部件抽屉；机体继续复用既有收藏模块。
- [x] 实现地形适配图标、评分和一句话说明。
- [x] 实现三套方案选择、保存和复制；可扩展命名编辑保留下一小版本。
- [x] 实现 20 秒试车场与自动返回工坊。

### 1.3 首批改装内容

- [x] 行走模块：公路轮、全地形轮、雪地履带、沙地气囊轮、浮航模块。
- [x] 炮弹：星脉、弹跳、冰霜、修复、闪电链、种子核心（策略与战斗差异已接入）。
- [x] 工具：修复臂、扫描雷达、牵引器、临时桥投射器（定义与解锁已接入）。
- [ ] 外观：3 个喷漆区、12 个贴纸、4 种车灯、4 种尾迹、4 种喇叭。
- [x] 炮弹双槽与战斗中键盘、手柄、触控快速切换。

### 1.4 四季山海垂直切片

- [ ] 建立一张包含山坡、河流、湖泊、沙滩和小岛的“山海谷”。
- [ ] 春夏秋冬四套地表、植被、天空、雾、声音和粒子。
- [ ] 每季至少一种路面物理差异和一条条件路线。
- [ ] 天气预报、关卡预警和推荐装备。
- [ ] 每季两个任务，共 8 个短任务。
- [ ] 两个环境 Boss：山岳巨像、机械蓝鲸。

### 1.5 阶段一验收

- [ ] 新玩家 90 秒内完成改装、试驾并进入任务。
- [ ] 五种行走模块在 20 秒试驾内有可观察差异。
- [ ] 六种炮弹拥有不同弹道、作用对象和反馈。
- [ ] 四季不仅换色，路线、物理、天气和任务均变化。
- [ ] Pad 横屏所有主操作触点至少 52px，无关键遮挡。
- [ ] 中端移动档同屏 45 个实体时保持目标帧预算。
- [ ] 原 v2 存档可无损升级。

## 阶段 2：水陆探索、环境任务与家庭合作

目标：从竞技场扩展为具有路线选择、世界反馈和亲子分工的冒险。

### 2.1 水陆与地貌

- [ ] 浅水阻力、深水浮航、潮汐、冰面冻结与破冰。
- [ ] 高度图/分块地形、LOD、实例化植被和碰撞简化。
- [ ] 瀑布、河流方向、水花、轮迹、泥浆、雪痕。
- [ ] 山洞、桥梁、索道、灯塔和可修复设施。

### 2.2 环境任务系统

- [ ] 任务图编排：触发、目标、步骤、失败回退和世界结果。
- [ ] 修复、救援、护送、探索、解谜、守护、摄影七类模板。
- [ ] 导电、冻结、推物、牵引、搭桥五种环境能力。
- [ ] 任务后展示装备贡献和替代路线。

### 2.3 世界记忆与图鉴

- [ ] 桥梁、灯塔、树木和机器人状态持久化。
- [ ] 四季自然图鉴、照片、发现地点和天气条件。
- [ ] 家园陈列室第一版。
- [ ] 伙伴机器人提示、收集、拍照和鼓励反馈。

### 2.4 亲子协作

- [ ] 双坦克模式继续支持键盘和两只手柄。
- [ ] 车组模式：驾驶与炮塔/工具分工。
- [ ] 导航员模式：第二屏或同屏地图与天气职责。
- [ ] 队友标记、快速指令、共同修复和协作机关。

### 2.5 阶段二验收

- [ ] 至少 12 个环境任务，单任务 3–6 分钟。
- [ ] 任务至少提供两种装备或路线解法。
- [ ] 世界变化退出并重新进入后仍然保留。
- [ ] 单人辅助和双人分工均能完成主线。
- [ ] 无必须依赖自由文本交流的合作节点。

## 阶段 3：世界地图、家园创造与安全分享

目标：形成长期内容框架和儿童创造空间，同时保持隐私安全。

### 3.1 世界地图

- [ ] 六个山海区域和区域进度。
- [ ] 故事季节与自由季节切换。
- [ ] 每日探索、周末家庭任务和离线活动日历。
- [ ] 区域守护者、稀有天气和隐藏路线。

### 3.2 家园与创造

- [ ] 可升级工坊、苗圃、陈列馆、灯塔和试车场。
- [ ] 家园装饰摆放、照片墙和坦克展台。
- [ ] 轻量关卡编辑器：地形块、桥梁、障碍、目标、天气。
- [ ] 本机家庭挑战和二维码/短码导入导出。

### 3.3 在线能力（独立开关）

- [ ] 家长入口和可验证同意流程。
- [ ] 家庭账号、云存档和冲突合并。
- [ ] 家庭/好友作品分享，无公开自由文本评论。
- [ ] 服务端每日挑战、回放校验和可信榜单。
- [ ] 昵称过滤、举报、删除数据和保留期限。

### 3.4 阶段三验收

- [ ] 六个区域均可通过配置注册，不修改核心循环。
- [ ] 儿童可以在 5 分钟内制作并本机试玩一个挑战。
- [ ] 未开启家长授权时不发送可识别数据。
- [ ] 在线服务不可用时完整离线玩法不受影响。

## 横向技术任务

### 性能

- [ ] 动态分辨率和 Battery/Balanced/High 三档。
- [ ] GPU/实例化粒子、对象池和空间哈希。
- [ ] glTF/纹理按区域懒加载，解码放入 Worker。
- [ ] WebGL 上下文丢失恢复和低内存回退。

### 输入与无障碍

- [ ] 技能拖拽指示、取消区和准星灵敏度。
- [ ] 手柄断连重连、按键映射和归属确认。
- [ ] 色觉模式、字幕、减少震动、减少闪光和大字模式。
- [ ] 左手布局和单手辅助模式。

### 质量门禁

- [ ] 类型检查、单元测试、生产构建。
- [ ] 桌面 1440×900、Pad 横屏 1180×820、手机竖屏 390×844。
- [ ] 键鼠、触控、单手柄、双手柄路径。
- [ ] 15 分钟性能采样和后台切回。
- [ ] 存档迁移、损坏 JSON、无痕模式和空间不足。
- [ ] 6–8 岁与 9–12 岁分组体验测试记录。
