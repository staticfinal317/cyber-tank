## 迭代 1：内容与手感量产（建议 3–4 周）

目标：把垂直切片扩成可持续玩 30 天的内容版本。

- [ ] 把 AI 条件分支重构为行为组件，新增护送、隐身、反弹、召唤、占点型敌人。
- [ ] 固定 60Hz Simulation Tick，增加碰撞空间哈希和对象池。
- [ ] 实现技能拖拽指示器、取消施法区、镜头边缘自动瞄准和灵敏度校准。
- [ ] 为三台机体增加独立主动技能、转向曲线、履带惯性和受击姿态。
- [ ] 每个主题增加 2 个专属事件、1 个区域 Boss、可破坏掩体和交互目标。
- [ ] 增加 20 个 3–6 分钟短任务，采用目标组合而非单纯清怪。
- [ ] 加入 GPU 粒子、InstancedMesh、移动端动态分辨率和画质自动降级。
- [ ] 增加程序化音乐分层、空间音效、触控振动和家长可控音量上限。
- [ ] 建立 6–8 岁、9–10 岁、11–12 岁三组可用性测试，每组至少 5 人。

验收指标：主流 2023 年后 Pad 稳定 60fps；中端手机 P95 帧时间小于 25ms；新玩家 60 秒内能独立移动和发射；8 岁组任务目标理解率大于 85%。

## 迭代 2：家庭账号与可信在线服务（建议 4–6 周）

目标：在不牺牲儿童隐私的前提下实现跨设备和社交成就。

- [ ] 家长入口、同意流程、儿童昵称白名单、数据导出与删除。
- [ ] 账号与云存档，加入 `SaveData` 迁移、冲突合并和离线队列。
- [ ] 服务端每日挑战、内容版本、签名规则和时区处理。
- [ ] 好友/家庭/班级排行榜；默认不开放陌生人互动。
- [ ] 回放改为输入帧并由服务端复算校验，排行榜只接受通过校验的成绩。
- [ ] 运营后台：主题轮换、挑战规则、奖励表和安全审计日志。
- [ ] 按发行地区完成儿童隐私、监护人同意和数据保留策略评审。

验收指标：弱网断线不丢进度；排行榜提交可追溯；未获家长同意的儿童账号不传可识别数据；删除请求可在策略时限内完成。

## 迭代 3：轻量“吃鸡”与多人协作（建议 6–8 周）

目标：扩展高复玩模式，但不引入强对抗和陌生人压力。

- [ ] “最后星核”改为 8–12 台玩家/机器人协作抢救撤离，核心目标是收集和护送。
- [ ] WebSocket/QUIC 房间服务、服务器权威 Simulation、输入预测与状态回滚。
- [ ] 动态安全区、空投、可驾驶载具、脚步/能量方向提示和观察模式。
- [ ] 家庭房间码、仅好友加入、无自由文本聊天、预设表情和快速指令。
- [ ] 段位采用成长徽章而非掉分惩罚；为低龄玩家加入机器人填充与动态难度。
- [ ] 蓝牙手柄按键映射、震动反馈、断连重连和双手柄归属确认。

验收指标：同地区网络 P95 输入延迟小于 120ms；重连后 5 秒内恢复；所有社交入口通过儿童安全审查；不付费也能完整参与核心模式。

## 发布门禁

每次版本发布必须完成：

1. `npm run check` 与 `npm run build`。
2. 桌面 1440×900、iPad 横屏 1180×820、手机竖屏至少三个真实浏览器截图。
3. 键鼠、单手柄、双手柄、触控四条操作路径。
4. 10 分钟性能采样、WebGL 上下文丢失恢复、后台切回和低电量模式。
5. 存档从上两个版本升级、空间不足、损坏 JSON 和无痕模式回归。
6. 儿童文案、奖励、付费入口和联网数据的专项产品审查。
