# ADR 0002：采用“远征工坊”融合视觉与装配式 Loadout

- 状态：已确认
- 日期：2026-07-15
- 视觉基线：[expedition-workshop-visual-target.png](../design/expedition-workshop-visual-target.png)

## 决策

采用一个连续的儿童远征准备流程，而不是把三个视觉方向平铺为三个互不相关的页面：

1. 以“山海世界与路线选择”建立出发期待。
2. 以“精密转台工坊”承担主要改装操作。
3. 以“拖拽/点击吸附、机械臂和伙伴机器人”承担 6–12 岁儿童的教学、反馈和陪伴。

工坊的主操作保持单一：`开始试驾`。`直接出发`为熟悉系统后的次要捷径。桌面支持拖动旋转与滚轮缩放；Pad/手机改用大触点分类、横向部件抽屉和固定可见的主操作。

## 技术影响

- `TankLoadout` 成为渲染、试驾、战斗与存档共享的唯一装备事实来源。
- 行走模块、炮弹与工具通过策略注册表扩展，新增内容不修改工坊控制器。
- 存档升级到 v3，保留 v2 高分、科技、星屑、排行榜与回放。
- 三套方案保存完整 Loadout；双炮弹槽可在战斗中用 Q、手柄 X 或触控按钮切换。
- 视觉仍使用当前 Three.js 管线和深海军蓝 / 青色 / 暖黄色令牌，避免创建第二套不兼容设计系统。

## 不采用的方案

- 不把路线地图、完整仓库、科技树和战斗 HUD 同时塞进工坊首屏。
- 不用纯静态背景假装 3D 改装；坦克、转台、机械臂、山体、水面和安装反馈均由实际 3D 场景渲染。
- 不将装备设计成单一数值强弱；每个部件必须对应不同地貌、任务或协作解法。
